# Contributing to speq-vscode-extension

## Workflow

- Create feature/fix branches from `main`.
- Keep UX changes aligned with CLI contracts.
- Test behavior against sample projects from `speq-examples`.

## Commit style

Use Conventional Commit prefixes (`feat:`, `fix:`, `docs:`, `chore:`).

## Pull request checklist

- [ ] Diagnostics and run flows tested manually.
- [ ] No runtime logic duplicated from CLI.
- [ ] Docs/screenshots updated if UI changed.
- [ ] CI checks pass.
